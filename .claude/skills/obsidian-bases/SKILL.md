# Obsidian Bases Skill

Use this skill when editing `.base` files or designing structured Obsidian views over note properties.

## Guidance

1. Keep property names aligned with the vault’s existing frontmatter schema.
2. Prefer clear filters and view names over dense implicit logic.
3. When building formulas or summaries, keep them readable enough for later manual edits in Obsidian.
4. Bases should reflect note metadata that already exists or is being introduced intentionally, not accidental property drift.
