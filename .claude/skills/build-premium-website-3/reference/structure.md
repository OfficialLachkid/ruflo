# Structure

## 1. Command Nav

- transparent at top
- collapses into sticky surface on scroll
- includes a thin progress line
- CTA should stay compact

## 2. Scene Hero

- full viewport minimum
- left side carries the main value proposition
- right or lower zone carries the signal-map
- trust chips sit under the lede
- image should feel cinematic, not stock-dashboard

## 3. Proof Ribbon

- one narrow band
- can include metrics, clients, or operating signals
- should visually reset the pace before longer sections

## 4. Perspective Split

- primary narrative section
- one side is copy
- one side is a sticky visual or image stack
- use reveal masks, not random fades

## 5. Capability Stack

- 4 core capability chapters
- one active at a time
- service copy should explain outcomes clearly
- keep panel architecture consistent

## 6. Showcase Frames

- three proof modules
- each one needs:
  - title
  - summary
  - visual frame
  - short caption or result line

## 7. Process Timeline

- 3 or 4 steps
- line should draw in
- active step should be visually obvious

## 8. Trust Matrix

- combine factual reassurance and a quote
- avoid giant logo walls
- use concise proof blocks

## 9. Contact Dock + Footer

- one clean CTA
- direct contact clarity
- compact form only if it improves the flow
- footer should remain structured and restrained
